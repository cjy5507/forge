Harness Engineering

Forge helps coding agents work like a structured software team.

It runs a virtual software company with guided delivery, session continuity, and runtime guardrails so build and repair work stays coordinated.

Build + repair flows
Tracked subagents

Session continuity
Virtual software company
Guided phases Intake → Discovery → Design → Develop → QA → Security/Fix → Delivery
Continuity across sessions Runtime state, tracked agents, and recovery context survive handoffs.
Evidence and guardrails Forge keeps specs, contracts, and code rules visible so review stays explicit when changes are ready to ship.

Control plane

  • Stop guard blocks premature exit while work is still active
  • Session start and end persist Forge runtime state
  • Subagent lifecycle stays visible in runtime state across handoffs

Quality gates

  • Code writes stay gated on required spec, design, rule, or contract inputs
  • Post-write reminders keep contracts and code-rule checks in view
  • Evaluation templates compare harness output against a baseline

Roles

  • CEO, PM, Researcher, CTO, Lead Dev, QA, Security, Troubleshooter
  • Progressive disclosure via references keeps prompts lean
  • Same plugin structure can ship to Claude and Codex
